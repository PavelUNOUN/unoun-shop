# UNOUN — Дорожная карта разработки
### Версия: 2.0 | Статус: Актуализирована под новую стратегию магазина

Эта дорожная карта больше не исходит из сценария "лендинг + форма заказа на главной". Теперь целевая модель проекта:

`лендинг -> корзина -> checkout -> онлайн-оплата -> личный кабинет -> повторные покупки`

Также в проекте закреплены:
- гостевой checkout без обязательной авторизации;
- приоритетная авторизация через ценность аккаунта;
- приветственные `500 ₽` бонусами только после входа;
- только `Яндекс Pay / Split` и полная онлайн-оплата;
- только `СДЭК ПВЗ` в MVP;
- хранение данных в инфраструктуре РФ;
- подготовка ассортимента под расходники и аксессуары;
- запуск контентного раздела `UNOUN Academy`.

## Правила процесса

- Дальнейшие кодовые изменения выполняются по новой дорожной карте, а не по старой landing-only логике.
- Сначала фиксируем архитектурное решение в документах, затем меняем код.
- Двигаемся отдельными небольшими этапами, которые можно проверить.
- Checkout, account, admin и Academy больше не считаются "дополнительными идеями", а являются частью основной целевой архитектуры проекта.
