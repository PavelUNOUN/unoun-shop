# UNOUN — исследование Ozon Delivery для собственного checkout

## Короткий вывод

Если цель такая:

- покупатель оформляет заказ на `unoun.shop`;
- вводит адрес на нашем checkout;
- сайт показывает ближайшие `Ozon` ПВЗ / постаматы;
- пользователь выбирает точку;
- мы создаем доставку через `Ozon`;

то по официальной документации, которую удалось подтвердить, это пока **не выглядит как готовый публичный сценарий через обычный Seller API Ozon**.

## Что Ozon официально дает

### 1. Seller API

Seller API у Ozon официально описан как API для продавцов, чтобы:

- управлять товарами;
- обновлять остатки и цены;
- работать с заказами;
- автоматизировать процессы внутри кабинета продавца.

Авторизация там строится через:

- `Client ID`
- `API key`

Их Ozon предлагает получать в личном кабинете продавца.

### 2. realFBS / доставка продавцом

В справке Ozon realFBS описан как модель, где:

- продавец хранит товар у себя;
- продавец сам отвечает за доставку;
- можно использовать любую подходящую службу доставки или доставлять самостоятельно.

Это значит, что `Ozon` в этом сценарии скорее маркетплейс и seller-интерфейс, а не готовый checkout-delivery API для стороннего сайта.

### 3. Настройка доставки в кабинете Ozon Seller

Официальная логика Ozon такая:

- вы настраиваете склады;
- вы настраиваете методы доставки;
- покупатель выбирает доступный способ доставки внутри оформления заказа на стороне `Ozon`.

## Что я не нашел в официальных источниках

Я **не нашел подтвержденного публичного API Ozon** для стороннего интернет-магазина, которое бы позволяло:

- по адресу покупателя получить ближайшие `Ozon` ПВЗ;
- получить стоимость / срок доставки для внешнего checkout;
- создать доставку вне заказа, оформленного на стороне `Ozon`.

Это важный блокер.

## Что это значит для нашего проекта

### Вариант A — собственный checkout на сайте

Для этого нам нужен API, который умеет:

1. искать ближайшие ПВЗ;
2. считать срок / стоимость;
3. создавать доставку;
4. получать статусы.

По подтвержденной документации `Seller API` этого сценария для внешнего checkout я пока не вижу.

### Вариант B — увести покупателя на Ozon

Если карточки уже есть на Ozon, рабочий сценарий может быть таким:

- пользователь на сайте нажимает `Купить на Ozon`;
- дальше оформляет заказ уже в checkout Ozon;
- выбор ПВЗ делает сам Ozon.

Это реалистичный путь, но это уже не наш собственный checkout.

### Вариант C — ручной / mock список ПВЗ

Технически мы можем сделать на сайте:

- поле адреса;
- примерные ПВЗ;
- UI под Ozon;

но без подтвержденного API это будет только витрина, а не реальная интеграция.

## Что нужно уточнить у поддержки Ozon

Нужно задать им прямой вопрос:

`Есть ли у Ozon публичный API или partner API для внешнего интернет-магазина, чтобы по адресу покупателя получать ближайшие Ozon ПВЗ/постаматы, стоимость и срок доставки, а затем создавать доставку вне заказа, оформленного на маркетплейсе Ozon?`

И отдельно:

`Можно ли использовать Ozon как службу доставки для checkout на собственном сайте, а не только внутри заказов Ozon marketplace?`

## Практический вывод на сейчас

Пока Ozon не подтвердит такой API, я бы не советовал переводить наш checkout на `Ozon-only delivery`, потому что есть высокий риск построить UI без боевого backend-сценария.
